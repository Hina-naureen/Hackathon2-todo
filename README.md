# Evolution of Todo — AI Powered Task Manager

> **Hackathon II** · Spec-Driven Development · Full-Stack · AI Agent · Docker

---

## Pitch

A task manager that understands you. Type *"add a task to call the doctor"* or *"delete task 3"* — the AI agent handles the rest. Built entirely through **Spec-Driven Development**: every line of code was generated by Claude Code from formal specifications. Zero manual coding. Five phases of evolution, from a Python console app to a cloud-native AI system.

---

## Features

- Sign up / sign in with JWT — route protection at the edge (Next.js middleware)
- Full task CRUD — persisted to Neon PostgreSQL via Alembic migrations
- AI chat assistant — natural language mapped to real database operations
- 5 AI tools: `create_task` · `list_tasks` · `update_task` · `delete_task` · `toggle_complete`
- Runs without an OpenAI key — local keyword simulation keeps the demo alive
- One-command Docker deploy

## Architecture

```
Browser
 ├─ Next.js :3000
 │   ├─ middleware.ts      JWT edge guard → redirect or pass
 │   ├─ /tasks             Task list + AI chat panel
 │   └─ /api/auth          Sign up / sign in
 │
 └─ FastAPI :8000
     ├─ /api/auth          Register · login → JWT
     ├─ /api/tasks         CRUD (JWT-protected)
     └─ /api/chat          AI agent
         └─ agent.py       OpenAI loop → tool dispatch
             └─ tools.py   5 tools → DBTaskStore → Neon PostgreSQL
```

**Key invariant:** Phase I `TaskManager` is untouched. The DB layer is injected via constructor — clean separation across all phases.

## Quick Start

```bash
git clone <repo-url>
cd Hackathon2-todo

# Fill in backend/.env and frontend/.env.local (see .env.example)

docker compose up --build
```

| URL | Service |
|-----|---------|
| `http://localhost:3000` | Frontend |
| `http://localhost:8000/docs` | Swagger API docs |
| `http://localhost:8000/health/db` | DB health check |

**Minimum env vars:**

```env
# backend/.env
DATABASE_URL=postgresql://...neon.tech/neondb?sslmode=require
BETTER_AUTH_SECRET=<64-char-hex>

BETTER_AUTH_SECRET=<same-value>
NEXT_PUBLIC_API_URL=http://localhost:8000
```

## Demo Flow

1. Open `http://localhost:3000` → redirected to sign-in
2. **Sign up** → JWT issued → land on `/tasks`
3. Type in the AI chat: `"add a task to review the slides"`
4. Watch the task appear in the list — no form filled, no button clicked
5. Type: `"show my pending tasks"` → AI lists them
6. Type: `"mark task 1 as done"` → status flips instantly
7. Type: `"delete task 2"` → gone
8. Visit `/health/db` → `{"status":"ok","dialect":"postgresql"}`

## Kubernetes Deployment (Phase IV)

**Prerequisites:** Minikube + Helm 3 + kubectl

```bash
# 1. Start Minikube and enable nginx Ingress
minikube start
minikube addons enable ingress

# 2. Point Docker to Minikube's daemon (PowerShell)
& minikube -p minikube docker-env --shell powershell | Invoke-Expression

# 3. Build images inside Minikube
docker build -t todo-backend:latest ./backend
docker build -t todo-frontend:latest ./frontend

# 4. Install — secrets are injected at install time, never committed to git
helm install todo-app ./helm/todo-app \
  --set secrets.databaseUrl="postgresql+psycopg2://user:pass@host.neon.tech/db?sslmode=require" \
  --set secrets.betterAuthSecret="$(python -c 'import secrets; print(secrets.token_hex(32))')" \
  --set secrets.openaiApiKey="sk-..."          # optional

# 5. Add /etc/hosts entry so todo.local resolves
echo "$(minikube ip)  todo.local" | sudo tee -a /etc/hosts
# Windows (run PowerShell as Administrator):
# Add-Content C:\Windows\System32\drivers\etc\hosts "$(minikube ip)  todo.local"

# 6. Open the app
# http://todo.local          ← Frontend
# http://todo.local/docs     ← Swagger API docs
# http://todo.local/health   ← Backend health
```

**Verify the deployment:**

```bash
kubectl get pods                          # all pods Running
kubectl get ingress                       # ADDRESS assigned
kubectl describe secret todo-app-secrets  # keys present, values redacted
curl http://todo.local/health             # {"status":"ok"}
curl http://todo.local/health/db          # {"status":"ok","dialect":"postgresql"}
```

**Upgrade after image rebuild:**

```bash
# Rebuild images, then:
helm upgrade todo-app ./helm/todo-app \
  --set secrets.databaseUrl="$DATABASE_URL" \
  --set secrets.betterAuthSecret="$BETTER_AUTH_SECRET" \
  --set secrets.openaiApiKey="$OPENAI_API_KEY"
```

**Teardown:**

```bash
helm uninstall todo-app
minikube stop
```
