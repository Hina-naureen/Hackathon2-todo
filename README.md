# Evolution of Todo - Phase I: Console App

**Hackathon II** | Spec-Driven Development & Cloud Native AI

## Overview

A command-line todo application built using Spec-Driven Development with Claude Code. All code was generated from specifications — no manual coding.

## Phase I Features

- Add tasks with title and description
- View all tasks in a formatted table
- Update task title or description
- Delete tasks with confirmation
- Toggle tasks between complete and pending

## Setup & Run

### Prerequisites
- Python 3.13+
- UV package manager (`pip install uv`)

### Install & Run

```bash
# Install dependencies
uv sync

# Run the app
uv run todo
```

## Spec-Driven Workflow Used

```
Constitution → Spec → Plan → Tasks → Implementation
```

All code generated by Claude Code based on specs. Zero manual coding.

## Development Approach

This project follows the Agentic Dev Stack:
1. Write Constitution (project principles)
2. Write Spec (what to build)
3. Write Plan (how to build it)
4. Write Tasks (breakdown)
5. Claude Code implements from tasks
