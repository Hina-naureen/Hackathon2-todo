# Evolution of Todo — AI-Powered Task Manager

> **Hackathon II** · Spec-Driven Development · 5 Phases · Full-Stack · AI Agent · Cloud-Native

---

## Pitch

A task manager that understands you. Type *"add a task to call the doctor tomorrow"* or *"delete task 3"* — the AI agent handles the rest. Built entirely through **Spec-Driven Development**: every line of code was generated by Claude Code from formal specifications. Zero manual coding. Five phases of evolution, from a Python console app to a cloud-native, event-driven system on Kubernetes.

---

## Features

- Sign up / sign in with JWT — route protection at the edge (Next.js middleware)
- Full task CRUD — persisted to Neon PostgreSQL via Alembic migrations
- AI chat assistant — natural language mapped to real database operations
- 5 AI tools: `create_task` · `list_tasks` · `update_task` · `delete_task` · `toggle_complete`
- Smart due-date parsing — *"call doctor on Friday"* → due date set automatically
- Dry-run preview — AI shows a confirmation card before creating a task
- Runs without an OpenAI key — local keyword simulation keeps the demo alive
- Domain events via Kafka + Dapr — every mutation emits `task.created/updated/deleted/toggled`
- One-command Minikube deploy · Automated DOKS (DigitalOcean) deploy script

## Architecture

```
Browser
 ├─ Next.js :3000
 │   ├─ middleware.ts         JWT edge guard → redirect or pass
 │   ├─ /tasks                Task list + AI chat panel
 │   └─ /api/auth             Sign up / sign in
 │
 └─ FastAPI :8000
     ├─ /api/auth             Register · login → JWT
     ├─ /api/tasks            CRUD (JWT-protected)
     │    └─ events.py        ──► Dapr sidecar :3500
     │                              └─► Kafka :9092
     │                                    └─► topic: task-events
     └─ /api/chat             AI agent
          └─ agent.py         OpenAI loop → tool dispatch
               └─ tools.py   5 tools → DBTaskStore → Neon PostgreSQL
```

**Key invariant:** Phase I `TaskManager` is untouched across all 5 phases. The DB layer is injected via constructor — clean separation of concerns.

7. Type: `"delete task 2"` → gone
8. Visit `/health/db` → `{"status":"ok","dialect":"postgresql"}`

## Kubernetes Deployment (Phase IV — Minikube)

**Prerequisites:** Minikube + Helm 3 + kubectl

```bash
# Automated (recommended)
bash scripts/deploy-minikube.sh   # reads backend/.env automatically

# Or manual:
minikube start --driver=docker
minikube addons enable ingress

# Point Docker to Minikube's daemon (PowerShell)
& minikube -p minikube docker-env --shell powershell | Invoke-Expression

# Build images inside Minikube
docker build -t todo-backend:latest ./backend
docker build -t todo-frontend:latest ./frontend

# Install — secrets injected at install time, never in git
helm install todo-app ./helm/todo-app \
  --set secrets.databaseUrl="postgresql+psycopg2://user:pass@host.neon.tech/db?sslmode=require" \
  --set secrets.betterAuthSecret="$(python -c 'import secrets; print(secrets.token_hex(32))')" \
  --set secrets.openaiApiKey="sk-..."    # optional

# Add hosts entry
# Linux/Mac:  echo "$(minikube ip)  todo.local" | sudo tee -a /etc/hosts
# Windows PS: Add-Content C:\Windows\System32\drivers\etc\hosts "$(minikube ip)  todo.local"
```

| URL | Service |
|-----|---------|
| `http://todo.local` | Frontend (Next.js) |
| `http://todo.local/docs` | Swagger API |
| `http://todo.local/health` | Health check |

**Verify:**

```bash
kubectl get pods                          # backend 1/1, frontend 1/1
kubectl get ingress                       # ADDRESS assigned
kubectl describe secret todo-app-secrets  # keys present, values redacted
curl http://todo.local/health             # {"status":"ok"}
```

**Teardown:**

```bash
helm uninstall todo-app && minikube stop
```

## Cloud Deployment (Phase V — DigitalOcean DOKS + Kafka + Dapr)

**Prerequisites:** `doctl` (DigitalOcean CLI) · Helm 3 · kubectl configured for DOKS cluster

```bash
# Set environment variables
export DATABASE_URL="postgresql+psycopg2://..."
export BETTER_AUTH_SECRET="$(openssl rand -hex 32)"
export OPENAI_API_KEY="sk-..."          # optional
export DOCR_REGISTRY="your-registry"   # DigitalOcean Container Registry name
export DOMAIN="todo.example.com"        # your domain
export IMAGE_TAG="v1.0.0"

# Full automated deploy (builds + pushes images, installs Dapr, deploys Kafka, installs chart)
bash scripts/deploy-doks.sh
```

The script:
1. Builds backend + frontend images and pushes to DOCR
2. Installs Dapr operator (`dapr-system` namespace)
3. Deploys single-node Kafka (KRaft, `k8s/kafka.yaml`)
4. Applies Dapr pub/sub component (`dapr/components/kafka-pubsub.yaml`)
5. Installs the Helm chart with DOKS values
6. Waits for all pods to be ready

**Event flow:**

```
POST /api/tasks (create task)
    └─► DB commit
    └─► events.py → Dapr sidecar :3500
                    └─► Kafka :9092 → topic: task-events
                                       └─► {"event_type": "task.created", ...}
```

Events are **fail-soft**: if Dapr is unavailable, a WARNING is logged and the API response is unaffected.

**Verify Kafka events:**

```bash
kubectl exec -it $(kubectl get pod -l app=kafka -o name) -- \
  /opt/kafka/bin/kafka-console-consumer.sh \
  --topic task-events \
  --bootstrap-server localhost:9092 \
  --from-beginning
```
